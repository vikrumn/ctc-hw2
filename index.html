Question #1

JavaScript is not an Object-Oriented Programming Language in the sense that there are no such things as classes, private variables or classical inheritance. Note that in the preceeding instructions, the word class means 'object'. Put all your JS code in index.js

  1. Define a 'Dog' Class
  2. Make a 'Dog' prototype method called 'bark' which takes in no parameter and returns 'woof'. All 'Dog' instances should have this method.
  3. Define another 'class' called 'ShibaInu' which calls the Dog constructor upon instantiation
  4. Have the 'ShibaInu' class inherit from 'Dog' using prototypal inheritance
  5. By doing the previous instruction, you've moved the 'Shiba Inu' object's pointer. Correct it by making ShibaInu.prototype.constructor point to the original 'ShibaInu' constructor.
  6. Make a prototype method from the ShibaInu class called 'bark' which takes in no parameter and returns 'much wow very javascript'

You should now be able to instantiate a ShibaInu object which returns 'much wow very javascript' upon calling myShibaInuObj.bark() Note: You can test your code at repl.it. You may also find this useful for JavaScript OOP
